Program.cs 

1->
var builder = WebApplication.CreateBuilder(args); -> builder is that preparation stagem Microsoft creates an object called builder. Inside it are things needed to build the application.
builder
│
├── Services
├── Configuration
├── Logging
├── Environment
└── Many More
args -> These are command-line arguments.

2-> 
builder.Services.AddControllers(); -> first Service Registration. ASP.NET must know which services exist. {"Hey ASP.NET...My application uses Controllers."} [ApiController] -> won't work. 
# builder.Services ->  Think of it as a big container (box). Everything that ASP.NET needs gets registered here.	

3-> 
builder.Services.AddEndpointsApiExplorer(); -> This is mainly for Swagger/OpenAPI. Scan and finds -> [HttpGet], [HttpPost], [HttpPut], [HttpDelete]. and collects information about them. Without it, Swagger doesn't know which APIs exist.

4-> 
builder.Services.AddSwaggerGen(); Generate Swagger documentation. This is Dependency Injection beginning to make sense.

5-> var app = builder.Build();
we actually build the application.  After this, you no longer register services. Because the application has already been built.

6-> 
if(app.Environment.IsDevelopment())
{
    app.UseSwagger();
    app.UseSwaggerUI();
}
Am I running in Development? if yes then use the swagger -> Companies don't expose Swagger publicly in Production.


7-> app.UseSwagger(); Creates the API documentation.

8-> app.UseSwaggerUI(); -> Creates the beautiful web page. Without this Swagger exists, but no UI.

9-> app.UseHttpsRedirection();  Suppose someone opens http://localhost ASP.NET automatically redirects

10-> app.UseAuthorization(); -> This is one Middleware. Every request passes through here. Later, JWT Roles Admin User All happen here.

11-> app.MapControllers(); -> Most imp line How does ASP.NET know PaymentController exists? -> It scans every Controller. and finds Every controler -> map them without every api Because ASP.NET doesn't know controllers exist.

12-> app.Run(); final line that starts kestrel 

13 ->  builder.Services.AddScoped<IPaymentService, PaymentService>(); //Whenever someone asks for an IPaymentService, give them a PaymentService.
